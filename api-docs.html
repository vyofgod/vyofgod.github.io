API Documentation

Integrate Osinters capabilities into your own applications and workflows with our powerful and flexible API.

Introduction

The Osinters API provides programmatic access to our core data collection, analysis, and visualization features. It's designed to be RESTful, predictable, and easy to use.

All API requests should be made to https://api.osinters.com/v1. All requests must be made over HTTPS. Calls made over plain HTTP will fail. API requests without authentication will also fail.

Authentication

Osinters API uses API Keys to authenticate requests. You can manage your API Keys in your account dashboard.

Your API Key carries many privileges, so be sure to keep it secret! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, and so forth.

Authentication to the API is performed via HTTP Bearer Authentication. Provide your API key in the Authorization header:

Authorization: Bearer YOUR_API_KEY

Rate Limiting

API requests are subject to rate limiting. The specific limits depend on your subscription plan. If you exceed the rate limit, you will receive an HTTP 429 Too Many Requests error.

Check the X-RateLimit-Limit, X-RateLimit-Remaining, and X-RateLimit-Reset headers in the API response to understand your current rate limit status.

Endpoints

Search Endpoint: /search

Method: POST

Description: Perform a comprehensive search across various data sources.

Parameters:

  • query (string, required): The search term or entity to investigate.
  • sources (array of strings, optional): Specific sources to query (e.g., ["social_media", "dark_web"]). Defaults to all available sources.
  • depth (integer, optional): The depth of the search. Defaults to 1.

Example Request:

curl -X POST https://api.osinters.com/v1/search \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
        "query": "example.com",
        "sources": ["domain_records", "social_media"],
        "depth": 2
      }'

Example Success Response (200 OK):

{
  "search_id": "s_123xyz",
  "status": "processing",
  "estimated_completion_time": "60s",
  "results_url": "https://api.osinters.com/v1/results/s_123xyz"
}

Data Formats

All API requests and responses are in JSON format. Ensure your Content-Type header is set to application/json for POST/PUT requests.

Error Codes

The API uses standard HTTP status codes to indicate the success or failure of a request:

  • 200 OK: Request successful.
  • 400 Bad Request: The request was unacceptable, often due to missing a required parameter or invalid JSON.
  • 401 Unauthorized: No valid API key provided.
  • 403 Forbidden: The API key doesn't have permissions to perform the request.
  • 404 Not Found: The requested resource doesn't exist.
  • 429 Too Many Requests: You've exceeded your rate limit.
  • 500, 502, 503, 504 Server Error: Something went wrong on Osinters's end. (These are rare).

Error responses will include a JSON body with more details:

{
  "error": {
    "type": "invalid_request_error",
    "message": "Missing required parameter: query"
  }
}
